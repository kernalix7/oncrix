# ONCRIX 검증 체크리스트

ONCRIX 운영체제를 위한 커밋 전 품질 보증 체크리스트입니다.

## 빠른 검증

```bash
cargo fmt --all -- --check && cargo clippy --workspace -- -D warnings && cargo build --workspace
```

---

## 1. 빌드 & 툴체인

- [ ] `cargo build --workspace` 에러 없이 성공
- [ ] `cargo clippy --workspace -- -D warnings` 경고 없이 통과
- [ ] `cargo fmt --all -- --check` 포맷 이슈 없음
- [ ] `cargo build --workspace` 성공
- [ ] 타겟 아키텍처 크로스 컴파일 빌드 성공

## 2. 보안

- [ ] 프로덕션 코드 경로에 `unwrap()` 또는 `expect()` 없음
- [ ] 모든 `unsafe` 블록에 불변 조건을 문서화하는 `// SAFETY:` 주석 포함
- [ ] `unsafe` 블록이 안전한 추상화로 감싸져 있고 전제 조건이 문서화됨
- [ ] 유저 스페이스 포인터가 커널 스페이스 역참조 전에 검증됨
- [ ] 소스 코드에 하드코딩된 자격 증명, 키, 비밀 없음
- [ ] 모든 시스템 경계에서 입력 검증 (시스콜, IPC 메시지, 드라이버 인터페이스)
- [ ] 특권 작업 전 능력(Capability) 검사 시행
- [ ] 경계 검사 없는 원시 포인터 산술 없음
- [ ] 인터럽트 핸들러에서 유저 스페이스 메모리 직접 접근 없음

## 3. 안정성

- [ ] 명시적 중단 조건 또는 타임아웃 없는 무한 루프 없음
- [ ] 정수 오버플로우 처리 (`checked_*`/`wrapping_*`/`saturating_*` 산술 사용)
- [ ] 인터럽트 컨텍스트 또는 인터럽트 비활성화 상태에서 패닉 없음
- [ ] 리소스 고갈에 대한 우아한 에러 처리 (OOM, FD 제한, 프로세스 제한)
- [ ] 모든 에러 경로에서 할당된 리소스 적절히 정리 (누수 없음)
- [ ] 교착 상태 없는 락 순서 유지 (락 계층 문서화)
- [ ] 커널 스레드에 대한 스택 오버플로우 보호

## 4. 개인정보 & 코드 위생

- [ ] 하드코딩된 파일 경로 또는 시스템 특정 경로 없음
- [ ] 코드 또는 주석에 개인 정보 또는 식별자 없음
- [ ] 프로덕션 코드에 디버그 출력문 없음
- [ ] 연관된 이슈 번호 없는 `TODO`/`FIXME` 없음
- [ ] 주석 처리된 코드 블록 없음 (삭제하거나 이슈 생성)

## 5. 성능

- [ ] 핫 패스에서 가능한 한 힙 할당 회피
- [ ] 락 보유 시간 최소화 (락을 잡은 상태에서 I/O 없음)
- [ ] IPC 메시지 크기 제한 및 검증
- [ ] 문서화된 정당화 없이 크리티컬 패스에 O(n^2) 이상의 알고리즘 없음
- [ ] DMA 버퍼가 하드웨어 요구사항에 맞게 정렬됨
- [ ] 스핀락은 짧은 크리티컬 섹션에서만 사용
- [ ] 메모리 매핑 I/O에 volatile 읽기/쓰기 사용

## 6. 문서화

- [ ] 모든 공개 타입, 트레이트, 함수에 `///` 문서 주석
- [ ] `#[cfg]` 어노테이션 설명과 함께 아키텍처별 동작 문서화
- [ ] 관련된 곳에 POSIX 호환성 메모 포함
- [ ] 공개 API 변경 시 README 및 docs 업데이트
- [ ] 모든 `unsafe` 추상화에 안전성 불변 조건 문서화
- [ ] 모듈 수준 `//!` 문서에 모듈 목적 설명

## 7. 코드 품질

- [ ] 에러 타입에 `Display`와 `Debug` 구현
- [ ] 미사용 코드 없음 (미사용 함수, 임포트, 변수)
- [ ] 타입 안전성 강화: 구별되는 개념에 뉴타입 사용 (PID, TID, FD, PhysAddr, VirtAddr)
- [ ] 매직 넘버보다 상수 사용
- [ ] 라이선스 헤더 포함: `// Copyright 2026 ONCRIX Contributors` + `// SPDX-License-Identifier: Apache-2.0`
- [ ] 모든 커널 스페이스 크레이트에 `#![no_std]` 사용
- [ ] Rust 명명 규칙 준수 (snake_case 함수, CamelCase 타입)
