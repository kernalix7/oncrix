# 보안 정책

## 지원 버전

| 버전 | 지원 |
|------|------|
| `main` 최신 | 예 |

## 취약점 신고

**보안 취약점에 대해 공개 이슈를 열지 마세요.**

보안 취약점은
[GitHub Security Advisories](https://github.com/kernalix7/oncrix/security/advisories)를
통해 신고해 주세요.

## 범위

### 범위 내

- 커널 코드의 메모리 안전성 문제
- 권한 상승 취약점
- IPC 능력(Capability) 우회
- 시스콜 인자 검증 실패
- 검증 없는 유저 스페이스 포인터 역참조
- 보안 영향을 미치는 정수 오버플로우
- 커널 동기화 프리미티브의 레이스 컨디션
- 드라이버 격리 우회

### 범위 외

- 하드웨어에 대한 물리적 접근이 필요한 취약점
- 소셜 엔지니어링 공격
- 업스트림 의존성의 이슈 (업스트림에 신고)
- 보안 영향 없는 서비스 거부

## 응답 일정

| 조치 | 시간 |
|------|------|
| 접수 확인 | 48시간 이내 |
| 초기 평가 | 7일 이내 |
| 심각한 이슈 수정 | 30일 이내 |

## 보안 관행

- 모든 커널 코드는 `#![no_std]` Rust로 작성
- 문서화된 안전성 불변 조건과 함께 최소한의 `unsafe` 사용
- 모든 IPC 엔드포인트에 능력(Capability) 기반 접근 제어
- 커널 접근 전 유저 스페이스 포인터 검증
- 의존성 취약점 확인을 위해 `cargo audit` 정기 실행
